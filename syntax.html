RackML Syntax

A RackML document must have an outer <racks> element.

<racks>

A <racks> element may contain any of the following attributes:

base
The base URL to use for links

The <racks> element contains zero or more <racks> elements.

<rack>

A <rack> element may contain the following attributes:

height
The height, in rack units, of this rack
name
The name of the rack. Displayed above the rack.

<rack> children

A <rack> element may contain any of the following elements:

<cables>
Represents a cable management system
<firewall>
Represents a firewall
<gap>
Empty space
<keyboard>
Represents a keyboard
<kvm>
Represents a KVM-switch
<monitor>
Represents a VDU
<patch>
Represents a patch panel
<pdu>
Represents a power distribution unit
<san>
Represents a SAN server
<server>
Represents a compute server
<switch>
Represents a networking switch
<tape>
Represents a tape drive
<ups>
Represents an uninterruptable power supply
Each of these elements may include the following attributes:
at
The lowest rack unit the device occupies.
height
The height, in rack units, of the device
href
A URL for additional information. The rack element will be clickable in the renderable SVG.
